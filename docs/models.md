# 支持模型

本站主要围绕 GPT API 接入、调用教程与 API 中转方案展开，当前支持与介绍的重点模型以 GPT 系列为主，适合开发者用于文本生成、代码辅助、图像生成等常见场景。

> API 平台地址：<https://jeniya.cn>

## 接口兼容说明

平台默认兼容 OpenAI 格式，开发者通常可以沿用 OpenAI SDK、请求结构和调用方式完成 GPT API 接入。  
如果你已经使用过 OpenAI 官方接口，通常只需要替换 `base_url` 和 `api_key` 即可完成对接。

## 当前重点支持的 GPT 模型

### GPT 文本模型

常见模型包括：

- gpt-5.5-pro
- gpt-5.5
- gpt-5.4-mini
- gpt-5.4
- 其他 GPT 文本模型

**适合场景：**

- AI 对话
- 内容生成
- 营销文案
- 代码生成
- 智能客服
- 知识库问答
- 自动化工作流

### GPT 图像模型

常见模型包括：

- gpt-image-2

**适合场景：**

- AI 生图
- 封面图生成
- 海报生成
- 社媒配图
- 电商素材图生成

## 模型选择建议

### 如果你是新手刚开始接入
建议优先使用：

- gpt-5.4-mini

优点是调用成本相对更低、速度较快，适合先完成联调和功能测试。

### 如果你需要更强的综合能力
建议优先使用：

- gpt-5.4
- gpt-5.5

适合正式业务、复杂问答、内容创作、代码辅助等场景。

### 如果你需要图像生成能力
建议使用：

- gpt-image-2

适合图片创作、视觉素材生成和内容配图场景。

## 选择模型时建议关注什么

在实际接入 GPT API 时，建议重点关注以下几个方面：

- 模型能力是否满足你的业务需求
- 响应速度是否适合线上场景
- 调用成本是否在预算范围内
- 是否需要图像生成等多模态能力
- 是否需要更稳定的中转接入方案

## 说明

- 支持模型会根据平台可用性和上游策略动态调整。
- 不同模型在价格、上下文长度、响应速度和能力上存在差异。
- 实际可用模型名称与调用方式，请以平台后台和接口文档为准。
